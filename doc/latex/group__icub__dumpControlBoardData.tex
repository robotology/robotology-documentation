\hypertarget{group__icub__dumpControlBoardData_config}{}\doxysubsection{How to config the application}\label{group__icub__dumpControlBoardData_config}
In case you want to personalize this dumper application you mainly need to edit the configuration files (conf/config.\+sh and conf/config-\/nodes.\+sh) and single parts configuration files (e.\+g. conf/icub\+Head.\+ini). 
\begin{DoxyItemize}
\item The config.\+sh file describes the part to be dumped (variable \$\+PARTS\+\_\+\+TO\+\_\+\+DUMP). 
\item The config-\/nodes.\+sh file describes the machines where the single modules (\mbox{\hyperlink{group__icub__controlBoardDumper}{control\+Board\+Dumper}} and yarpdatadumper) will be executed. In particular, the \$\+NODE\+\_\+\+DUMPERS variable defines the machines where data will be dumped (usally in the directory \$\+NODE\+\_\+\+DUMPERS\+:/tmp/icub/data\+Dumper). 
\item The icub\+Head.\+ini file is the file used by the \mbox{\hyperlink{group__icub__controlBoardDumper}{control\+Board\+Dumper}} to dump the head joints (see similar files for the other robot parts). This file can be used to define the joints to be dumped, the sampling time and the type of data to be dumped (see the \mbox{\hyperlink{group__icub__controlBoardDumper}{control\+Board\+Dumper}} details). 
\end{DoxyItemize}
