In this tutorial we show how to write the XML documentation of an \mbox{\hyperlink{namespaceiCub}{i\+Cub}} module.\hypertarget{xml_module_documentation_sec_xml_intro}{}\doxysection{Introduction}\label{xml_module_documentation_sec_xml_intro}
The convenience of providing the documentation of an \mbox{\hyperlink{namespaceiCub}{i\+Cub}} module in the XML format is twofold\+: (1) it standardizes the way we declare relevant information for the module (arguments, dependencies ...) and serves thus to automatically generate the Doxygen documentation; (2) the XML file accompanying the module is indeed the description file itself used by the \href{http://wiki.icub.org/yarpdoc/yarpmanager.html}{\texttt{ yarpmanager}}.\hypertarget{xml_module_documentation_sec_xml_head}{}\doxysection{Heading}\label{xml_module_documentation_sec_xml_head}
The XML file should start with the following lines that contain the instructions on how to produce the Doxygen documentation from the XML itself\+:


\begin{DoxyCode}{0}
\DoxyCodeLine{<?\textcolor{keyword}{xml} \textcolor{keyword}{version}=\textcolor{stringliteral}{"{}1.0"{}} \textcolor{keyword}{encoding}=\textcolor{stringliteral}{"{}ISO-\/8859-\/1"{}}?>}
\DoxyCodeLine{<?\textcolor{keyword}{xml-\/stylesheet} \textcolor{keyword}{type}=\textcolor{stringliteral}{"{}text/xsl"{}} \textcolor{keyword}{href}=\textcolor{stringliteral}{"{}yarpmanifest.xsl"{}}?>}

\end{DoxyCode}


The description is all contained within the {\bfseries{module}} section\+:

\DoxyCodeLine{</\textcolor{keywordtype}{module}>}

\end{DoxyCode}
